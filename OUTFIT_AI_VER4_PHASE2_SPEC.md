# OUTFIT AI Ver4 第2フェーズ 改善仕様書

作成日: 2026-07-13  
対象基準: Ver4 第1フェーズ（commit `849e965`）  
文書状態: 2026-07-13 実装・ローカル検証済み

## 1. 第2フェーズの目的

第2フェーズでは、マスター登録をさらに短く、迷いにくく、楽しい体験へ改善する。

中心方針は次のとおり。

> 最初はサッと登録して使い始め、運用しながら情報を育てる。

詳細情報をすべて入力しないと使えない管理アプリにはしない。写真と少ない選択だけで登録を完了でき、商品名、型番、購入情報、タグ写真、靴箱などは後から追加できる構成とする。

## 2. 維持する制約

- GitHub Pagesで公開する静的PWA
- HTML、CSS、Vanilla JavaScriptのみ
- React、Vue、npm、ビルド環境は使用しない
- OpenAI等の生成AIは使用しない
- クラウド保存、ログイン、複数ユーザーは導入しない
- ブランドサイト画像は使用しない
- ユーザーが撮影・選択した画像だけを使用する
- 既存localStorageデータを壊さない
- 既存機能を維持し、リファクタリングは必要最小限にする

## 4. 基本UX原則

### 4.1 クイック登録を主役にする

新規登録では、最初に最低限必要な項目だけを見せる。

最低限の登録項目:

- カテゴリ
- 写真
- メイン色
- 種類
- 名称

ブランドは候補を表示するが、未設定でも保存可能とする。

詳細情報は「あとで情報を育てる」へ格納する。

### 4.2 自動確定しない

写真解析で得た色、名称、タグ等は候補として表示する。ユーザーが選んだものだけを確定する。

### 4.3 自由入力を残す

ブランド、種類、名称は候補選択だけに限定しない。「その他」または自由入力を常に利用可能にする。

### 4.4 登録後すぐ使える

詳細情報が空でも、次を利用できる状態にする。

- 今日のおすすめ
- SVGフィギュア
- 着用履歴
- カルテ
- 情報充実度

## 8. 色判定改善

### 8.1 現状の問題

現在は画像全体の平均RGBを使用しているため、次の影響を受けやすい。

- 床、壁、机などの背景
- ハンガー
- 影
- 照明の色
- 白飛び
- 靴底など面積の異なる部位

### 8.2 新しい解析方式

生成AIや外部ライブラリを使わず、Canvas上で次の処理を行う。

1. 画像を64×64pxへ縮小
2. 四隅から背景色候補を取得
3. 四隅の色に近い画素の重みを下げる
4. 画像中央70%の画素を強く評価
5. 透明画素を除外
6. 極端な白飛び・黒つぶれ画素の重みを下げる
7. RGBをHSVへ変換
8. 色ファミリーごとの出現量を集計
9. 上位3色を候補として返す
10. 候補ごとの比率を確信度として表示

### 8.3 色ファミリー

- 白
- 黒
- グレー
- ネイビー
- 青
- 水色
- ベージュ
- ブラウン
- グリーン
- カーキ
- 赤
- ワイン
- ピンク
- イエロー
- オレンジ
- パープル
- その他

既存の色値はそのまま利用可能とし、新色だけを追加する。

### 8.4 表示仕様

```text
写真から近い色を見つけました

● ネイビー　68%
● 黒　　　　21%
● グレー　　11%

実物に近い色を選んでください
```

- 上位3候補をカラーチップ付きボタンで表示
- 1位は初期選択するが、未確定表示にする
- ユーザーがタップした時点で確定
- 「別の色を選ぶ」から全色一覧を開ける
- 色解析に失敗した場合は全色一覧を表示

### 8.5 メイン色とサブ色

第2フェーズでは次の構造を追加可能とする。

```js
{
  color: "ネイビー",
  secondaryColor: "白",
  colorConfidence: 68,
  colorSource: "photo"
}
```

`secondaryColor`は任意。既存アイテムでは空文字を補完する。

## 18. SVGコーデフィギュア改善

### 18.1 目的

- 色合わせを一目で判断できる
- 今日画面を毎日開きたくなる
- 写真が未登録でもコーデの雰囲気が分かる
- 外部画像や生成AIを使わずに着せ替える

### 18.2 デザイン方針

- 現在より丸みのある、かわいいデフォルメ体型
- 頭身は約3〜3.5頭身
- 顔、髪、手を自然な形へ改善
- 輪郭線を少し柔らかくする
- 軽い影とハイライトを付ける
- 靴が見分けやすい立ち姿にする
- スマートフォンで主役として見えるサイズを確保
- CSSアニメーションは軽い登場演出だけにする

### 18.3 レイヤー構造

SVGを次の論理レイヤーで構成する。

1. 影
2. 脚・肌
3. パンツ
4. 靴
5. 胴体
6. シャツ
7. 腕・手
8. 頭・顔・髪
9. 柄
10. ハイライト

画像ファイルを増やさず、JavaScriptで属性やSVGパスを切り替える。

### 18.4 シャツ反映

- 色
- 半袖、長袖、七分袖
- ワイシャツ: 襟と前立て
- ポロシャツ: ポロ襟と短いボタン
- Tシャツ: 丸首
- 開襟シャツ: 開いた襟
- ゆったりフィット: 胴幅を広げる
- スリムフィット: 胴幅を狭める

### 18.5 柄反映

第2フェーズでは簡易表現とする。

- 無地: 追加なし
- ストライプ: 細い縦線
- チェック: 縦横線
- ドット: 小さな丸を規則配置
- ボーダー: 横線
- その他: 柄を表示せず色だけ反映

柄はSVGの`pattern`または`clipPath`を使い、服の範囲外へ出さない。

### 18.6 パンツ反映

- 色
- スラックス: 細めで直線的
- チノパン: 標準形
- デニム: ステッチ表現
- ワイドパンツ: 幅を広げる
- カーゴパンツ: サイドポケット
- ショートパンツ: 丈を短くする
- クロップド: 足首を見せる

### 18.7 靴反映

- 色
- ローファー: 甲の帯
- プレーントゥ・ビジネス靴: 細長い形
- スニーカー: 厚めのソールと靴紐
- ランニングシューズ: 反ったつま先
- スリッポン: 靴紐なし
- ブーツ: 足首まで覆う
- サンダル: 肌が見える
- レインシューズ: 高めの筒

### 18.8 フォールバック

種類が未設定またはその他の場合:

- シャツは標準長袖トップス
- パンツは標準ロングパンツ
- 靴は標準シューズ

不正な色値の場合はグレーを使用する。

### 18.9 アクセシビリティ

- SVGへ`role="img"`
- 現在の組み合わせを`aria-label`で説明
- 例: 「白の長袖ワイシャツ、ネイビーのスラックス、黒のローファー」
- 色だけで種類を表現しない

## 19. データ構造変更

### 19.1 アイテム追加フィールド

```js
{
  secondaryColor: "",
  colorConfidence: null,
  colorSource: "manual",     // manual | photo
  collar: "未設定",
  fit: "未設定"
}
```

パンツでも`pattern`を使用可能にする。

### 19.2 既存データ移行

`normalizeState`で不足フィールドを補う。

```js
{
  secondaryColor: "",
  colorConfidence: null,
  colorSource: "manual",
  collar: "未設定",
  fit: "未設定",
  pattern: existingPattern || "未設定"
}
```

- 保存キー`outfit-ai-pwa-v2`は変更しない
- 既存の`color`、`subtype`、`brand`をそのまま維持
- ブランド正規化は新規保存・編集保存時だけ実行
- 既存ブランドを読み込み時に勝手に書き換えない

## 20. エラー・空状態

### 写真解析失敗

```text
写真から色を絞り込めませんでした。
実物に近い色を選んでください。
```

全色一覧を表示し、登録は継続可能。

### 写真なし

```text
写真はあとから追加できます。
種類と色を選んで登録してください。
```

### 候補外ブランド

自由入力を許可する。エラーにしない。

### 名称が空

色と種類から名称を自動生成する。それでも生成できない場合だけ入力を促す。

### 保存容量不足

現在の既存メッセージを維持するが、写真圧縮後の保存失敗であることを分かりやすく表示する。

## 21. 情報充実度

クイック登録直後に低すぎる印象を与えないよう、必須情報と育成情報を分ける。

推奨配点:

| 項目 | 点数 |
|---|---:|
| 写真 | 20 |
| 名称 | 10 |
| 色 | 10 |
| 種類 | 10 |
| ブランド | 10 |
| 種類固有情報 | 10 |
| タグ | 5 |
| 商品名・型番 | 10 |
| 購入情報 | 5 |
| メモ | 5 |
| タグ写真 | 3 |
| 靴箱 | 2 |

合計100点。

クイック登録で写真・名称・色・種類・ブランドが揃えば60%となり、「登録できた」感を持たせる。

## 22. 推薦ロジックへの追加影響

第2フェーズでは大規模変更をせず、新しい種類・柄を少しだけ利用する。

追加候補:

- ポロシャツ＋スニーカー＋休日: 加点
- ワイシャツ／ドレスシャツ＋革靴＋仕事・営業: 加点
- スラックス＋ローファー／ビジネス靴: 加点
- デニム＋スニーカー: 加点
- 柄物が2点以上ある場合、ベーシックカラー靴を加点
- 長時間歩く予定でランニング／ウォーキングシューズ: 加点
- 雨でレインシューズ: 加点

各追加ルールの点数は実装前に一覧化し、100点へ張り付きすぎないよう既存重みも調整する。

## 23. 受け入れ条件

### 色判定

- 写真選択後に色候補が最大3件表示される
- 候補ごとに色チップと割合が表示される
- ユーザーが候補を変更できる
- 全色一覧から選択できる
- 解析失敗でも登録を継続できる
- 背景色だけを拾う割合が現在より低下する

### 種類・ブランド

- カテゴリ別の種類候補が表示される
- カテゴリ別のブランド候補が表示される
- 候補外の自由入力ができる
- `REGAL`、`SKECHERS`、`adidas`、`NIKE`が靴候補にある
- `UNIQLO`、東京シャツ、AOKI、洋服の青山、ORIHICAがシャツ候補にある
- ORIHICA、AOKI、洋服の青山、EDWINがパンツ候補にある

### 柄

- シャツ柄にドットがある
- 既存の無地、ストライプ、チェック、プリントを維持する
- パンツにも柄を設定できる
- 柄未設定の既存データを正常表示できる

### クイック登録

- 詳細欄を開かず保存できる
- 写真選択後4操作程度で保存できる
- ブランド未設定でも保存できる
- 名称候補をそのまま採用できる
- 保存後すぐ今日画面で使用できる

### SVGフィギュア

- シャツ、パンツ、靴の色が反映される
- 半袖と長袖を区別できる
- ワイシャツ、ポロシャツ、Tシャツを見分けられる
- スラックス、デニム、ワイド、ショートを見分けられる
- ローファー、ビジネス靴、スニーカー、ブーツを見分けられる
- ドット柄を簡易表示できる
- 未設定データでも崩れない
- スマートフォン幅で画面からはみ出さない

### 互換性

- Ver4第1フェーズの保存データを読み込める
- 既存写真、履歴、メンテナンス、靴箱を維持する
- Service Worker更新後に新UIが表示される
- GitHub Pages上で主要機能を操作できる

## 27. 完成条件

第2フェーズは次の状態をもって完成とする。

- 写真から実物に近い色を現在より選びやすい
- 色を自動確定せず、候補から簡単に選べる
- シャツ、パンツ、靴の種類をすぐ選べる
- よく使うブランドが最初から候補にある
- ドット柄を登録できる
- 写真選択後15〜30秒程度で保存できる
- 詳細情報を後から育てられる
- SVGフィギュアで種類と色合わせを直感的に確認できる
- 既存のVer4データと機能を壊さない
- GitHub Pagesの公開URLで最終確認済み

最終的なUX判断基準は「入力項目が増えたか」ではなく、「登録までの迷いと操作が減ったか」とする。
